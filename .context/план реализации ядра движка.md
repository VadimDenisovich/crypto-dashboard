# План реализации ядра торгового движка

## Контекст

Проект — асинхронный криптотрейдинговый движок для курсовой работы.
Архитектура: Clean Architecture (Domain → Application → Infrastructure),
event-driven, общение между сервисами только через PostgreSQL и Redis Pub/Sub.

Текущая директория `trade-engine-crypto/` пустая (есть только `.context/`).
Задача — заложить ядро движка: Domain и Application слои + unit-тесты.
Инфраструктурный слой (CCXT-адаптеры, БД, Redis) появится позже отдельно.

Источник правил — `.context/CLAUDE.md`. Ключевое:
- В `domain/` запрещены импорты `ccxt`, `sqlalchemy`, `aioredis`, `fastapi`,
  `pydantic` и любых инфра-библиотек.
- Только `asyncio`-код, никаких блокирующих вызовов.
- DI через конструктор; адаптеры приходят аргументами.
- Mocking внешних сервисов в тестах обязателен.

## Принятые архитектурные решения (подтверждены пользователем)

1. **Доменные модели**: `@dataclass(frozen=True, slots=True)` + `decimal.Decimal`
   для всех денежных полей. Никакого Pydantic в `src/domain/`.
2. **Стратегия владеет историей**: внутренний `collections.deque(maxlen=N)`
   создаётся в `Strategy.__init__`. Сигнатура hot-path —
   `on_candle(candle: Candle) -> Signal | None`.
3. **MarketDataProvider** — отдельный интерфейс в `domain/interfaces.py`.
   Нужен, чтобы `StrategyRunner` не был жёстко завязан на `asyncio.Queue` и
   чтобы будущий `BacktestEngine` мог подсунуть `HistoricalMarketDataProvider`.

## Шаг 2 — Domain Layer

`src/domain/enums.py`:
- `class Side(StrEnum): BUY = "buy"; SELL = "sell"`
- `class OrderType(StrEnum): MARKET, LIMIT`
- `class OrderStatus(StrEnum): NEW, OPEN, PARTIAL, FILLED, CANCELED, REJECTED, EXPIRED`
- `class TimeFrame(StrEnum): M1="1m", M5="5m", M15="15m", H1="1h", H4="4h", D1="1d"`

`src/domain/models.py` — все frozen+slots dataclass'ы:
- `Candle(symbol, timeframe, timestamp: datetime, open, high, low, close, volume)`
- `Signal(strategy_name, symbol, side, price: Decimal | None, size: Decimal, metadata: Mapping[str, Any])`
- `Position(symbol, side, entry_price, size, current_pnl)`
- `Order(order_id, symbol, side, type, price, size, filled_size, status, fee)`
- `Balance(currency, free, used, total)`
- Валидация инвариантов (например, `high >= low`, `size > 0`) в `__post_init__`.

`src/domain/exceptions.py`:
- `DomainError(Exception)` — корень.
- `RiskRejectedError(DomainError)` — с полем `reason: str`.
- `InsufficientBalanceError(RiskRejectedError)`
- `LotSizeError(RiskRejectedError)`
- `InvalidSignalError(DomainError)`
- `OrderExecutionError(DomainError)`

`src/domain/interfaces.py` — `abc.ABC` с async-методами:
- `ExchangeAdapter.fetch_ohlcv(symbol, timeframe, since, limit) -> list[Candle]`
- `ExchangeAdapter.create_order(symbol, side, type, size, price=None) -> Order`
- `ExchangeAdapter.get_balance() -> Mapping[str, Balance]`
- `MarketDataProvider.subscribe(symbol, timeframe) -> AsyncIterator[Candle]`
- `Strategy` — атрибуты `name, symbol, timeframe`; абстрактный
  `on_candle(candle) -> Signal | None` (sync — расчёт индикаторов CPU-bound,
  не блокирует loop надолго).
- `EventBus.publish(channel: str, payload: Mapping[str, Any]) -> None` (async).

**Deliverable**: 4 файла + `__init__.py`. Пауза.

## Шаг 3 — Application Layer

`src/application/events.py` — константы каналов:
`NEW_TRADE`, `BALANCE_UPDATE`, `ENGINE_STATUS`, `STRATEGY_ERROR`.

`src/application/risk_manager.py`
```
class RiskManager:
    def __init__(self, adapter: ExchangeAdapter,
                 max_position_pct: Decimal,
                 min_lot: Decimal): ...

    async def check(self, signal: Signal) -> Signal:
        # 1. достать баланс через adapter.get_balance()
        # 2. проверить, что quote-баланс покрывает size * price
        # 3. проверить, что size >= min_lot
        # 4. проверить, что size * price <= max_position_pct * total_equity
        # → InsufficientBalanceError / LotSizeError / RiskRejectedError
```

`src/application/order_executor.py`
```
class OrderExecutor:
    def __init__(self, adapter: ExchangeAdapter, event_bus: EventBus): ...

    async def execute(self, signal: Signal) -> Order:
        # adapter.create_order(...) с try/except → OrderExecutionError
        # при успехе: event_bus.publish(NEW_TRADE, order_payload)
```

`src/application/strategy_runner.py`
```
class StrategyRunner:
    def __init__(self, strategy: Strategy,
                 mdp: MarketDataProvider,
                 risk: RiskManager,
                 executor: OrderExecutor,
                 event_bus: EventBus): ...

    async def run(self) -> None:
        async for candle in self.mdp.subscribe(strategy.symbol, strategy.timeframe):
            signal = self.strategy.on_candle(candle)
            if signal is None:
                continue
            try:
                approved = await self.risk.check(signal)
                await self.executor.execute(approved)
            except RiskRejectedError as e:
                await self.event_bus.publish(STRATEGY_ERROR, {...})
            except OrderExecutionError as e:
                await self.event_bus.publish(STRATEGY_ERROR, {...})
            # цикл не падает на одной плохой свече
```

## Шаг 4 — Тесты

Тулинг: `pytest` + `pytest-asyncio` (`asyncio_mode="auto"`) +
`unittest.mock.AsyncMock`.

`tests/application/test_risk_manager.py`:
- happy path — баланс достаточен, лот валиден → возвращает сигнал.
- баланс недостаточен → `InsufficientBalanceError`.
- size < min_lot → `LotSizeError`.
- size × price > max_position_pct × equity → `RiskRejectedError`.

`tests/application/test_order_executor.py`:
- успех: `adapter.create_order` вернул Order → `event_bus.publish(NEW_TRADE)`
  вызван ровно один раз с правильным payload.
- адаптер кинул исключение → `OrderExecutionError`, событие не публикуется.

`tests/application/test_strategy_runner.py`:
- мокаем `MarketDataProvider.subscribe` как `async def → AsyncIterator`,
  скармливаем 3 свечи; `Strategy.on_candle` возвращает Signal на 3-й.
- проверяем: `RiskManager.check` вызван 1 раз, `OrderExecutor.execute` вызван 1 раз.
- сценарий с `RiskRejectedError`: executor НЕ вызван, но цикл живёт и
  опубликовался `STRATEGY_ERROR`.

`tests/domain/test_models.py`:
- инвариант `high >= low` → `ValueError` при создании `Candle`.
- `Signal.size <= 0` → `ValueError`.

**Deliverable**: дерево созданных файлов + результат `pytest -v`. Пауза для код-ревью.
